About Our Project Implementation

  • Problem Statement :

    • The branch elections in college are usually carried out through moodle making it less transparent and centralized.Not just elections but any polling which happens in our class can sometimes favor a particular group.Develop a system that could solve these floating concerns,keeping the polls anonymous but transparent and must follow all the rules.

  • Flow:

    • The address that deploys the smart contract sets the candidates contesting for the election while deploying the smart contract. This address becomes the chairperson (or admin) and has access to grant access to vote. Their address is used to grant the access, unless which he/she can’t provide their votes. The admin can conclude the election by ending it.

  • Features Implemented :
    • Admin
      • Give addresses the access to vote
      • End the election
      • Can see the candidates vote count
      • Can Vote only once
    • Voters
      • Can see the candidates vote count
      • Can Vote only once
    • User With No Access To Vote
      • Can see the candidates vote count
      • Can see the form for the election, but cannot vote
    • UI
      • Live/ complete tag near Election Results based on election status (sets to complete when admin “ends the election”)
      • Form Visibility to select the candidates based on whether the voter has voted/yet to vote.
      • When connected to admin wallet, an extra section to “add voters” & “end election”
